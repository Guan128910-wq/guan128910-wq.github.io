由于我后面有多服务器的需求，所以顺便部署一个服务器状态面板。

---

我选择的软件是[XUGOU](https://github.com/zaunist/xugou)，因为我的域名本身就是挂载在CloudFlare上的，安装起来也方便，但是需要注意，第一种部署方式也就是一键部署用不了，因为作者填了自己的数据库，所以只有作者能部署成功，如果在我的Cloudflare上，那是永远都成功不了的（笑）：

![](https://r2.guange.top/%E9%83%A8%E7%BD%B2%E4%BA%86%E6%9C%8D%E5%8A%A1%E5%99%A8%E7%8A%B6%E6%80%81%E9%9D%A2%E6%9D%BF_1.webp)

所以我们需要使用Fork部署模式，在XUGOU的Github界面点击右上角的Fork创建，然后在wrangler.toml中修改自定义域名和数据库名字即可，在[triggers]前面添加以下代码：

```bash
routes = [
  { pattern = "设置访问的域名", custom_domain = true }
]
```

::: tip
可以在[我自己的Fork](https://github.com/Guan128910-wq/xugou)中查看修改后的样子
:::

记得先在Cloudflare中创建一个[D1 SQL数据库](https://dash.cloudflare.com/435b9b83f67a2ecc9d683c10aaa10969/workers/d1)，藏在存储和数据库的二级菜单里：

![](https://r2.guange.top/%E9%83%A8%E7%BD%B2%E4%BA%86%E6%9C%8D%E5%8A%A1%E5%99%A8%E7%8A%B6%E6%80%81%E9%9D%A2%E6%9D%BF_2.webp)

修改完之后去到Cloudflare的[Workers界面](https://dash.cloudflare.com/435b9b83f67a2ecc9d683c10aaa10969/workers-and-pages)，导入一个git存储库就行：

> ![](https://r2.guange.top/%E9%83%A8%E7%BD%B2%E4%BA%86%E6%9C%8D%E5%8A%A1%E5%99%A8%E7%8A%B6%E6%80%81%E9%9D%A2%E6%9D%BF_3.webp)
这是导入成功的样子

成功之后访问域名，登录账户之后点击客户端监控，将一键安装命令复制至终端即可一键安装。

::: info
默认账户：admin；默认密码：admin123
:::

安装完毕之后，输入以下命令即可一键启动：

```bash
Xugou-agent start
```

![](https://r2.guange.top/%E9%83%A8%E7%BD%B2%E4%BA%86%E6%9C%8D%E5%8A%A1%E5%99%A8%E7%8A%B6%E6%80%81%E9%9D%A2%E6%9D%BF_4.webp)

::: info
如果遇到无法启动的问题，可尝试将一键安装命令中的undefined改为自己部署的域名（带http或https）
:::
