# 安装vitepress

我找到的主题是基于[vitepress](https://vitepress.dev/zh/)的，所以我需要先部署，用[以前的方法](https://blog.guange.top/posts/2025-09-13.html)部署了1Panel之后，去到仓库界面，复制下载链接：

![](https://r2.guange.top/%E6%9B%B4%E6%8D%A2%E5%8D%9A%E5%AE%A2%E5%BC%95%E6%93%8E_1.webp)

去到终端界面，将源码克隆至服务器：

```bash
git clone 你复制的链接
```
这里的链接不一定要SSH，HTTPS也可以，之后打开Node.js部署界面，新建一个环境：

![](https://r2.guange.top/%E6%9B%B4%E6%8D%A2%E5%8D%9A%E5%AE%A2%E5%BC%95%E6%93%8E_2.webp)

然后定位到项目的目录，一般在root文件夹下，然后选择preview作为启动命令，之后在防火墙将端口放行之后，在日志中获取应用端口，将应用端口和放行端口输入之后，就可以成功访问了。

![](https://r2.guange.top/%E6%9B%B4%E6%8D%A2%E5%8D%9A%E5%AE%A2%E5%BC%95%E6%93%8E_3.webp)

# 自定义

主题到手肯定是要修改的，这里详细讲一下[gitlak](https://github.com/gitalk/gitalk/)，首先要在[github](https://github.com/gitalk/gitalk/)创建一个仓库，需要保持公开，建议设置一个md：

![](https://r2.guange.top/%E6%9B%B4%E6%8D%A2%E5%8D%9A%E5%AE%A2%E5%BC%95%E6%93%8E_4.webp)

创建好之后，我们需要在右上角设置的开发者工具里面获取cilentid和cilentsecret，创建一个OAuth，然后填入有关信息：

![](https://r2.guange.top/%E6%9B%B4%E6%8D%A2%E5%8D%9A%E5%AE%A2%E5%BC%95%E6%93%8E_5.webp)

Application name写仓库名字，Homepage URL和Authorization callback URL写自己的域名（带http或https）就行了，创建完毕之后要点一下Client secrets右边的按钮才能显示。

# 最后工作

根据github的信息更改相应文件，一般只需要修改avatar和config文件，记得按照posts文件夹里的规范撰写博客。我整理了一个压缩包去描述规范（仅适用于[本主题](https://github.com/Alittfre/vitepress-theme-bluearchive)，去到其他主题可能会有兼容性不同），新建文件夹里的内容需要根据[这里](https://vitepress-theme-bluearchive.vercel.app/)一起参考。

[下载地址](https://openlist.guange.top/@s/812ASZjp)
