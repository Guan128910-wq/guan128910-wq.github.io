# 选择GPT-SoVITS的原因

原因很简单，开源，速度相对其他的本地TTS软件更快一点，安装也非常简单，最重要的是我中意的模型就是基于它训练的（略略略）

# 安装

我找好的模型是SLNeil大佬训练的，简介里有下载地址：

[视频链接](https://www.bilibili.com/video/BV1o4fyYuEPW/)

@[bilibili](BV1o4fyYuEPW)

之后去到[GPT-SoVITS的github界面](https://github.com/RVC-Boss/GPT-SoVITS/tree/main)根据指引下载程序，注意下载的程序会占较大空间，需要保证硬盘空间足够。

下载解压完之后，直接双击go-webui.bat就会弹出一个网页，这就证明已经安装成功了：

![](https://r2.guange.top/%E8%AF%95%E7%94%A8%E4%BA%86GPT-SoVITS_1.webp)

::: warning
如果发生报错，请自行[访问官网](https://www.yuque.com/baicaigongchang1145haoyuangong/ib3g1e/pgah3gvetrdy8ryt)获取信息
:::

# 使用

[GPT-SoVITS](https://www.yuque.com/baicaigongchang1145haoyuangong/ib3g1e)需要源语音才能正常使用，所以需要获取你要TTS的模型的原语音，我这里因为是使用蔚蓝档案学生的语音，所以直接从这两个链接里下载学生的源语音就行了：

::: details
[链接1](https://kivo.wiki/)  [链接2（不稳定但可以看语音长度）](https://schaledb.com/student)
:::

下载完语音之后，将模型导入程序文件夹，然后在端口9874的网页中点击第二个大项，再打开推理中的TTS推理WebUI即可：

> ![](https://r2.guange.top/%E8%AF%95%E7%94%A8%E4%BA%86GPT-SoVITS_2.webp)
图片上有点击顺序

::: info
GPU目前只支持带CUDA的英伟达显卡，如果没有，就会用CPU来推理（AMD显卡也不行）
:::

等待程序初始化之后，会弹出这个网页：

![](https://r2.guange.top/%E8%AF%95%E7%94%A8%E4%BA%86GPT-SoVITS_3.webp)

然后将参考音频（3-10s）上传，输入文本然后生成即可。

::: tip
这里建议拿个好点的设备，因为我只有A卡，所以推理只能用cpu，很慢，11代i5生成两秒钟（11个字）的音频，5次轮询竟然要17秒才能生成。
:::

[点击这里可以查看轮询次数不同的生成区别](https://www.123865.com/s/N9Qhjv-0NhNh)

说实话，这两个我都不是很满意，所以我又用日语重新生成了一遍，[点击这里就能听](https://www.123865.com/s/N9Qhjv-KNhNh)，果然效果好了不少，这个模型应该就是基于日文进行训练的。

在右边还可以添加参考音频，我尝试上传了几个，[点击这里可以查看生成结果的区别](https://www.123865.com/s/N9Qhjv-uNhNh)，结果就是偶尔会有些许提升，初此之外没什么太大的区别。

::: info
仓库里面的文件，jp/cn是语言，加了e前缀+数字的是生成的语音+轮询次数，加了e后缀的是副参考音频（对于生成的语音就是参考音频数）
:::

# 总结

试了一圈发现效果还不错，可以作为简单的TTS使用，不过想要让它达到更好的效果，就要看模型作者的更新和软件的优化了，我打算到时候买一个小主机部署这个项目给大家跑，大家帮我记住这个饼，有钱了我一定办（笑）
