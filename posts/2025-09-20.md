# 更换服务器

我的思路是更换一个服务器，只部署WordPress，并使用OpenLiteSpeed达到速度最大化，再启用IPV6，使用CloudFlare的AAAA模式进行转发，达到双线节流，买服务器和Cloudflare的部署可以在[这里](https://blog.guange.top/posts/2025-09-13.html)查看。

# 备份服务器

由于OpenResty和OpenLiteSpeed不同，故无法使用1Panel的备份功能，我在网上找到用插件备份的方法，具体操作是使用[UpdraftPlush](https://wordpress.org/plugins/updraftplus/)（可直接在WordPress应用商店搜索到），安装完之后，先到插件的设置界面添加云存储：

![](https://r2.guange.top/%E4%BC%98%E5%8C%96%E4%BA%86%E6%88%91%E7%9A%84%E5%8D%9A%E5%AE%A2%E7%BD%91%E7%AB%99_1.webp)

备份完毕后，下次只需要登录同样的账号就能恢复备份了。

# CyberPanel安装（弃用）

OpenLiteSpeed目前只有一个[CyberPanel面板](https://cyberpanel.net/)是用原生的，所以在选择好系统后（2.5美元的只能选Alma）使用XShell连接服务器，并输入以下代码：

```bash
sh <(curl https://cyberpanel.net/install.sh || wget -O - https://cyberpanel.net/install.sh)
```

![](https://r2.guange.top/%E4%BC%98%E5%8C%96%E4%BA%86%E6%88%91%E7%9A%84%E5%8D%9A%E5%AE%A2%E7%BD%91%E7%AB%99_2.webp)

::: tip
单域名2G内存以下机器可以装LiteSpeed EnterPrise，但是要到官网申请，如果嫌麻烦可以直接安装OpenLiteSpeed。
:::

![](https://r2.guange.top/%E4%BC%98%E5%8C%96%E4%BA%86%E6%88%91%E7%9A%84%E5%8D%9A%E5%AE%A2%E7%BD%91%E7%AB%99_3.webp)

::: tip
这里除非你有使用远程MySQL的需求，否则不建议安装
:::

安装完之后，就可以通过8090端口进入CyberPanel面板了，但是我这里遇到了500错误，所以就用[Vultr自带的LiteSpeed WordPress](https://docs.litespeedtech.com/cloud/images/wordpress/#__tabbed_1_2)部署了。

# 使用存储桶

由于我的服务器设置在国外，所以需要使用存储桶来加快加载速度，我这里使用的是[CloudFlare R2](https://www.cloudflare.com/zh-cn/developer-platform/products/r2/)，10GB免费存储和超大AB类访问量，足够应付我的小博客了。
不过管理文件的软件不知道为什么，找的几个都报错，有了解的大佬可以在评论区答疑。

# 图片压缩

图片体积过大也是加载速度慢的原因之一，我这里使用[Imagine](https://github.com/meowtec/Imagine/releases)将图片压缩，在80%的质量下，可以减小20％-60％的图片体积。
